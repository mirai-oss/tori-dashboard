# Larkから指示して日報を動かす（コマンドBot設定ガイド）

グループで **「日報」「週報」「月報」「再送」** と発言すると、Botがレポートを再生成して配信します。
「日報が届いてない → グループで『再送』と打つ → すぐ届く」が実現します。

仕組み:
```
Lark で「再送」と発言
   │  ①イベント購読（Larkが発言を通知）
   ▼
GAS Webアプリ（新規）──②内容を判定──▶ GitHubのレポート起動 ──▶ 日報がグループに届く
   └─────────── ③「了解しました。生成中です」と即返信
```

---

## 前提: Larkアプリの「ボット機能」を有効化（ここが関門）

以前 `234007 App does not enable bot feature` で止まった箇所です。まずここを通します。

1. https://open.larksuite.com/app → 対象アプリ（App ID: `cli_aac75df6e1b81e18`）を開く
2. 左メニュー **「機能」→「ボット」** を開き、**ボットを有効化**（管理者承認が必要な場合は承認まで）
3. 有効化できたら次へ。※どうしても有効化できない場合はここで教えてください（別方式を検討します）

---

## ステップ1: 権限（スコープ）を追加

アプリの **「権限管理」** で以下を追加 → 「バージョンを作成して公開」（管理者承認が要る場合あり）:

- **メッセージの送受信**（`im:message` 系。単聊・グループのメッセージ取得と送信）
  - 探す時のキーワード: 「メッセージ」「message」

---

## ステップ2: GAS（受け口）を作る

1. https://script.google.com → **新しいプロジェクト**（※ダッシュボード用とは別の新規プロジェクト）
2. リポジトリの [`gas/LarkBot.gs`](gas/LarkBot.gs) の内容を貼り付け → 保存
3. **⚙ プロジェクトの設定 → スクリプトプロパティ** に4つ登録:

   | プロパティ | 値 |
   |---|---|
   | `GH_TOKEN` | GitHubのPAT（Actions: Read and write。日報トリガーで使ったものでOK） |
   | `LARK_APP_ID` | `cli_aac75df6e1b81e18` |
   | `LARK_APP_SECRET` | アプリのSecret |
   | `LARK_VERIFY_TOKEN` | （ステップ4で発行される値。今は空でOK、後で入れる） |

4. **デプロイ → 新しいデプロイ → 種類: ウェブアプリ**
   - 実行ユーザー: **自分**
   - アクセスできるユーザー: **全員**（匿名を含む。Larkがアクセスするため必須）
   - デプロイ → **ウェブアプリURL**（`https://script.google.com/macros/s/.../exec`）をコピー

---

## ステップ3: 送信テスト（先に片方向を確認）

エディタで関数 `testDispatchDaily` を実行 → ログに `true` が出て、GitHub Actionsが動けばOK
（初回は承認ダイアログが出るので許可）。

---

## ステップ4: Larkでイベント購読を設定

1. アプリ管理画面 → **「イベント購読」**
2. **リクエストURL** に、ステップ2でコピーした**ウェブアプリURL**を貼る
   - Larkが自動でURL検証します（このコードは検証に自動応答します）→「成功」になればOK
3. 画面に表示される **Verification Token** をコピーし、
   GASのスクリプトプロパティ `LARK_VERIFY_TOKEN` に貼り付けて保存
4. **「暗号化(Encrypt Key)」は空のまま**にしてください（暗号化すると受け取れません）
5. **イベントを追加** → **「メッセージを受信 `im.message.receive_v1`」** を追加して保存

---

## ステップ5: グループにBotを入れて動作確認

1. 日報が届くグループの設定 → メンバー追加 → **このアプリ（Bot）を追加**
2. グループで **「@Bot 再送」** または Botへのダイレクトメッセージで **「日報」** と送信
   - グループでは基本 **@メンション** が必要です
3. Botが「了解しました。日報を再生成しています…」と返し、1〜2分で日報カードが届けば **完成** 🎉

---

## 使えるコマンド

| 送る言葉 | 動作 |
|---|---|
| `日報` / `再送` | 日報を再生成して配信 |
| `週報` | 直前の完了週の週報を配信 |
| `月報` | 前月の月報を配信 |
| `ヘルプ` | 使い方を表示 |

> ※発言はキーワードで判定します。将来ここに **Claude（AI）** を挟めば、
> 「昨日の売上は?」等の自然文にも答えるエージェントに拡張できます（フェーズ2）。

---

## 困ったとき

- URL検証が失敗 → デプロイのアクセスが「全員（匿名を含む）」か、URLが `…/exec` で終わっているか確認
- Botが無反応 → グループで@メンションしているか、`im.message.receive_v1` を追加したか、権限公開したか
- 「生成中」は出るが届かない → GitHub Actionsのログで「カードを送信」を確認（トークン権限）

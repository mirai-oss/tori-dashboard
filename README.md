# 鳥一代グループ 経営ダッシュボード

飲食店グループの売上・入金・口コミ・広告をリアルタイムで見られる社内向けダッシュボードです。

## ファイル構成

| ファイル | 役割 |
|---|---|
| `index.html` | ダッシュボード本体（開くだけで動きます） |
| `app.js` | ロジック（index.html から読み込まれます） |
| `sample-data.js` | サンプルデータ（API未接続時のデモ用） |
| `gas/Code.gs` | スプレッドシート側に貼る API コード（Google Apps Script） |

`index.html` をブラウザで開けば、まずサンプルデータで全機能を試せます。

---

## 1. 権限とログイン

権限は4種類。**アカウント（ID・パスワード・権限・担当店舗）はスプレッドシートの「アカウント」シートで管理**します。ダッシュボード内の「アカウント管理」タブ（社長・本部のみ表示）からも発行・編集できます。

| 権限 | 見える範囲 | 機能 |
|---|---|---|
| **社長** | 全店舗 | 全機能＋アカウント発行 |
| **本部** | 全店舗 | 全機能＋アカウント発行 |
| **マネージャー** | 担当店舗（複数可） | 担当店舗の比較・広告管理あり |
| **店舗** | 自店のみ | 他店比較・広告管理・アカウント管理は非表示 |

### デモアカウント（API未接続時のみ）

## 2. スプレッドシート連携のセットアップ（リアルタイム反映）

### 手順

1. データの入っているスプレッドシートを開く
2. メニュー **拡張機能 → Apps Script** を開く
3. `gas/Code.gs` の中身を貼り付けて保存
4. 右上 **デプロイ → 新しいデプロイ → 種類: ウェブアプリ**
   - 「次のユーザーとして実行」: **自分**
   - 「アクセスできるユーザー」: **全員**（URLを知っている社内の人だけが使う前提。実際の認証はID/パスワードで行われます）
5. 発行された `https://script.google.com/macros/s/…/exec` の URL をコピー
6. ダッシュボードの **⚙ 接続設定** に URL を貼り付け → 「接続テスト」→「保存」
7. ログインし直すと、シートの「アカウント」で認証され、データがリアルタイム取得されます

初回アクセス時に「**アカウント**」「**接続設定**」シートが自動作成されます。

### リアルタイム反映のしくみ（＋軽量化）

- ダッシュボードは設定した間隔（既定60秒、30秒〜5分で変更可）で自動的にシートを再取得します
- タブを切り替えて戻ってきた時も自動更新されます
- スプレッドシートを編集 → 最大1分程度でダッシュボードに反映
- ヘッダーの「↻ 更新」でいつでも手動更新できます

**表示・通信を軽くする工夫（Code.gsを最新にすると有効）:**
- **ログインは待たせない** … 認証が済んだ時点で画面を表示し、重いデータ読込は裏で実行（「認証中…」が一瞬になります）
- **変更検知でムダ読みを削減** … 自動更新のたびに全データを読むのではなく、まず軽い「署名（各シートの行数＋最終更新時刻）」だけ取得し、**前回から変化がある時だけ**フル取得します
- **読み込む期間を制限** … 接続設定で「直近◯ヶ月」を選ぶと、その期間ぶんだけ取得して転送量・描画を軽くできます（既定24ヶ月。前年比には13ヶ月以上必要。深い過去分析をする時だけ「全期間」に）
- ※ これらは `gas/Code.gs` を最新版にして再デプロイすると有効になります。古いままでも従来通り全件取得で動作します（自動フォールバック）。

### データシートの形式（1行目がヘッダー）

| キー | 既定シート名 | 必要な列（別名も自動認識） |
|---|---|---|
| `daily`（必須） | 分析_日別店舗 | 店舗名(店舗), 日付(営業日), 純売上(売上) ／ 任意: 総客数(客数), 仕入(仕入金額), 人件費合計, 現金 |
| `media` | 分析_媒体別日次 | 店舗名, 営業日(日付), 媒体(媒体名), 人数, 純売上 |
| `deposit` | 入金DB | 店舗(店舗名), 日付(営業日), 入金額(入金合計) |
| `review` | 口コミ推移ログ | 店舗名, 平均星 ／ 任意: 取得日, 累計, 前回比 |
| `ad` | 広告DB | 日付, 店舗名, 媒体, 広告費(費用/金額) |

シート名が違う場合は「接続設定」シートの2列目を実際の名前に書き換えるだけでOKです。
列の順番は自由で、ヘッダー名は部分一致＋別名(カッコ内)で自動認識します。

> **取り込みに失敗したとき**: 画面上部に赤い「⚠ 実データを取り込めていません」バナーが出て、どのシートが「シート未受信」「列が見つかりません」なのかを一覧表示します。そこを見て、接続設定のシート名か、対象シートの1行目の列名を直してください。

## 3. 後からデータソースを追加する（広告管理など）

**スプレッドシートに「`DB_`」で始まるシートを作るだけ**で、次回同期時から自動でダッシュボードに配信されます。

- `DB_広告`（列: 日付・店舗名・媒体・広告費）→ 「広告管理」タブが自動で有効になり、店舗別・媒体別の広告費と売上対比（広告費率）を表示
- その他の `DB_○○` シート → 「広告管理」タブ内に汎用テーブルとして表示され、CSVエクスポートにも含まれます

「接続設定」シートに行を足して登録する方法でも同じことができます。

## 4. 主な画面

- **ダッシュボード** — 日次/週次/月次/年間/期間指定のKPI（売上・原価率・人件費率・FL・客数・客単価・口コミ）、売上チャート、FL内訳、媒体別売上。
  - **全店/合算を選ぶと**「全店比較」（全店舗を常時表示、行クリックで店舗切替）
  - **店舗を選ぶと**「日別明細」（期間内の1日ごとの 売上・客数・客単価・前年比・累計差異（対前年）・FL を日付順に表示。月次なら1日〜月末）
- **推移分析** — 売上/客数/客単価 × 日別/週別/月別 × 合計/店舗別/媒体別の折れ線、前年重ね表示
- **PL（損益）** — **月次／年間／期間指定**を切り替えて、**店舗ごとのPL**（売上高→原価→粗利→人件費・広告費・経費→営業利益、前期/前年同期比較つき）を表示。売上・原価・人件費は分析_日別店舗、広告費はDB_広告から**自動連携**。家賃・水道光熱費などの経費はスプレッドシートに **`DB_PL`** タブ（列: `年月 / 店舗名 / 費目 / 金額 / メモ`、1行=1費目、店舗名空欄=全社共通）を作って入力すると自動反映。全店表示では店舗別の営業利益比較も。
- **入金管理** — 月送り（‹ ›）で月を選び、**1日ごとの入金予定（現金売上）・入金（ATM）・未入金・累計未入金**と状態バッジを表示。全店表示時は店舗別サマリーも
- **広告管理** — 広告シート接続で自動有効化（上記参照）
- **口コミ** — Google評価の推移（直近12ヶ月）と最新スナップショット。スプレッドシートに「ダイニーDB」（来店アンケート）があれば、**「また来たいと思いますか？」の店舗別平均**（直近30日/累計・月次推移）も表示し、ダッシュボードのKPIにも「ダイニー再来店意向」カードが出ます。**同じ実店舗に複数のGoogleリスティングがある場合、親店舗のタブに別名店舗も並べて表示**します（例：黒霧屋 新横浜 → カラオケ彩・うお蔵 新横浜も表示）。対応関係は `app.js` の `REVIEW_CHILDREN` で定義（店舗が増えたら追記）。
- **AI検索** — 質問文から店舗・期間・媒体・指標を読み取り、ダッシュボード内のデータだけで回答（外部送信なし）。例：「鳥一代 本店 今年のランチの単価は？ 推移も」。期間＝今日/今週/先週/今月/先月/◯月/今年/去年、指標＝売上・客数・単価・原価率・人件費率・FL・利益・口コミ・未入金、「推移」でグラフ表示。
- **アカウント管理**（社長・本部のみ）— アカウントの発行・編集・無効化・削除

## 5. CSV / PDF ダウンロード

- **⬇ CSV** — 今見ているページの全テーブルを1つのCSV（Excel対応・BOM付きUTF-8）でダウンロード
- **🖨 PDF** — 印刷ダイアログが開くので「PDFとして保存」を選択。操作ボタン類は消え、タイトル・期間・出力者入りのレポートとして出力されます

## 6. セキュリティについて（社内利用前提）

- 閲覧にはログインが必須。トークンは12時間で失効し、ログイン失敗が続くと一時ロックされます
- パスワードは「アカウント」シートに保存されるため、**スプレッドシート自体の共有範囲は管理者のみ**にしてください
- ダッシュボード（index.html 一式）は社内のファイル共有や限定公開のホスティングに置いてください。GASのURLとID/パスワードが揃わない限りデータは取得できません

## 補足

- サンプルデータの入金DBはダミーのため、日次の現金売上と金額が一致せず未入金の数字が不自然に見えます。実データ接続後は正しく照合されます。
- 全店比較の店舗一覧は「daily シートに登場する店舗」から自動生成されるので、新店舗はシートにデータを入れるだけで自動的に追加されます。

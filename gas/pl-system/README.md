# PL管理システム（Google Sheets + GAS）

飲食店グループの月次PL（損益）を管理するスプレッドシートを、コンテナバインドの Google Apps Script で自動構築・運用するシステム。
ダッシュボード用スプレッドシート（サーバー）と双方向連携する。

> **開発メモ**: このシステムは複数の Claude セッション（Cowork / mirai 環境）から並行開発されている。
> **`pl_system.gs` がマスターソース。** 変更したら必ずこのリポジトリを更新すること。

## デプロイ方法

1. PL管理スプレッドシートを開く → 拡張機能 → Apps Script
2. `pl_system.gs` の中身をコード.gs に全文貼り付けて保存
3. シートに戻り、メニュー **「▶ PL管理」→「システム再構築（初期化）」** を実行（約2分）
   - ⚠ まれに「スプレッドシートのサービスに接続できなくなりました」で途中終了する（Google側の一時エラー）。**もう一度メニューから再実行すれば直る**
4. 初回のみ「🔄 取込_日別」タブ A1 の「アクセスを許可」を押す（IMPORTRANGE許可）

## タブ構成（buildAllで9タブを再生成）

🏠 ガイド / ⚙ 設定 / 📚 科目マスタ / ✍ 販管費入力（唯一の手入力タブ）/ ✅ 入力チェック / 📋 月次PL / 📈 年間推移 / 🔄 取込_日別（隠し）/ 🔄 取込_月次（隠し）

⚠ 再構築するとタブは作り直される（gidが毎回変わる）。販管費入力の既存データはダッシュボードDB_PLから再シードされる。

## 主要機能

### 1. 期間指定（最大24ヶ月）
- ⚙ 設定 `B3`=期間開始月, `B4`=期間終了月（yyyy/mm）
- 月リストは `D8:D31` に自動生成され、月次PL・年間推移・取込_月次がこれに追従

### 2. 媒体販促費の自動計上 ★2026/07追加
特定媒体（デリバリー等）の純売上 × 設定率 を「媒体販促費（自動）」（区分O）として自動計上する。

- **対象媒体リスト: ⚙ 設定 `H12:H21`（黄色セル・10枠）** — 現在: Ring-style / いちご屋 / Live GATE / Peevo。**空き枠に媒体名を追記するだけで対象に追加される（コード変更・再構築不要）**
- **料率: ⚙ 設定 `D5`**（現在 24.0%）
- データフロー:
  1. `取込_日別 K1` — ダッシュボード「分析_媒体別日次」から QUERY+IMPORTRANGE で 年月×店舗×対象媒体の純売上を集計（FILTER+COUNTIF で設定H12:H21 のリストと照合）
  2. `取込_月次 H列`（媒体販促対象売上）— K1結果を SUMIFS で月次×店舗に展開
  3. `月次PL/年間推移` — 科目マスタB56「媒体販促費（自動）」（データ源=自動｜媒体販促）が H列×D5 を自動計算
  4. `DB_PLへ転記` — `computeAutoPromoRows()` が金額>0の行を生成し、メモ `媒体販促費（自動計上）` 付きでダッシュボードDB_PLへ洗い替え転記
- ⚠ IMPORTRANGEのセル数上限対策のため、K1は B列/J:K列/O列 を分割読込して横結合している（A2:Q100000 の一括読込は「結果が大きすぎます」エラーになる）

### 3. DB_PLへ転記
メニュー「▶ PL管理 → DB_PLへ転記」で、✍販管費入力の手入力行＋媒体販促費（自動）行をダッシュボードの `DB_PL` タブへ転記。
- 対象月の既存行は差し替え（洗い替え）、それ以外の月は保持
- 手入力行と自動計上行は別々の月キーで管理（`syncToDbPl()`）

## 開発時の注意

- 科目マスタの行番号は固定依存あり: **B56 = 媒体販促費（自動）**（月次PLでは65行目）
- `AUTO_MEMO = '媒体販促費（自動計上）'` がDB_PL自動転記行の識別マーカー。変更しないこと
- gviz CSV で検証する場合は `gid=` を使うこと（シート名に絵文字が入っているため `sheet=` は効かない）。gvizはキャッシュされるのでキャッシュバスター推奨
- 転記完了アラートに「手入力 N行＋媒体販促費（自動）M行」と件数が出るので検証に使える
